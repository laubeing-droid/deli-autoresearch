# Spec-Driven Rules

1. No implementation without approved requirements, design, tasks, and acceptance criteria.
2. Execute one Task ID at a time.
3. Never modify requirements while implementing.
4. If implementation reveals a specification defect:
   - stop;
   - record evidence;
   - set status to REDESIGN;
   - do not silently change the target.
5. All completion claims require evidence files.
6. Natural-language summaries are not evidence.
