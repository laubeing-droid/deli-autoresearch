# SPEC-290 Requirements

## Goal

Generate release artifacts documenting the exact proof boundary of the Unified Legal Kernel.

## Requirements

1. **R1** — Generate theorem manifest listing all proven theorems and their status.
2. **R2** — Generate axiom report listing all axioms used (standard only, zero custom).
3. **R3** — Generate runtime conformance report showing pipeline alignment.
4. **R4** — Generate allowed claims (what can be stated based on proofs).
5. **R5** — Generate forbidden claims (what cannot be stated).
6. **R6** — Generate release boundary report.
7. **R7** — Mark Banach track as non-blocking.
8. **R8** — Human review of release language.
9. **R9** — Release tag requires human decision.

## Non-goals

- Actual release tagging (human decision required).
- Repository visibility changes (human decision required).
